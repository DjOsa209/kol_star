## 1. 产品背景与目标

### 1.1 背景

当前全球媒体、KOL、创作者、代理商等合作资源分散在不同项目 Excel、个人文档、区域团队资料、邮件和聊天记录中，缺少统一沉淀、持续维护和复用机制。HQ 难以掌握全球资源全貌，区域团队也难以快速判断资源质量、历史合作效果和风险状态。

主要问题包括：

- 资源分散，缺少统一资源库和标准字段。
- 资源标签、评分、等级口径不统一，难以比较和筛选。
- 资源质量依赖人工经验判断，筛选效率低。
- 合作效果没有形成闭环，优质资源难以复用，风险资源可能被重复使用。
- 缺少智能助手，项目人员无法基于自然语言需求快速获得推荐名单。
- 缺少治理规则中心，数据完整性、更新频率、评分逻辑、推荐规则和预警规则难以统一管理。

### 1.2 产品定位

平台定位为面向全球市场、品牌、公关、增长、区域团队和 HQ 管理人员的媒体/KOL/创作者资源智能运营系统。

一句话概括：

> 把“靠人找资源、靠 Excel 管资源、靠经验判断资源”升级为“统一沉淀、规则治理、智能推荐、数据复盘”的资源运营系统。

### 1.3 产品目标

- 建立全球统一资源库，集中管理媒体、KOL、KOC、创作者、代理商等合作资源。
- 建立标签和评分评级体系，使资源可筛选、可比较、可排序、可复用。
- 建立智能资源助手，支持自然语言项目需求解析、资源推荐、推荐理由、预算建议和风险提醒。
- 建立项目协作流程，覆盖推荐、候选确认、合作执行、效果回填和复盘。
- 建立治理规则配置中心，支持评分权重、等级阈值、必填字段、更新频率、推荐规则和预警规则配置。
- 建立数据看板，帮助 HQ 和区域团队查看资源分布、资源质量、合作效果和运营风险。

### 1.4 成功指标

| 指标类型 | 成功标准 |
| --- | --- |
| 资源资产 | 资源主档、平台账号、标签、评分、等级、状态、负责人等信息可统一维护 |
| 推荐效率 | 项目人员输入需求后，可以获得结构化需求解析和可解释推荐清单 |
| 项目闭环 | 推荐资源可以加入项目，合作后可以回填效果数据并影响评分 |
| 治理质量 | 必填字段、更新频率、推荐规则、预警规则可配置并对资源状态产生影响 |
| 管理视角 | HQ 可查看全球资源总量、区域分布、等级占比、更新率、风险和项目效果 |
| 研发交付 | 产品、设计、研发、测试可基于本文档拆分功能任务和测试用例 |

## 2. 用户角色与权限

### 2.1 用户角色

| 角色 | 说明 | 核心诉求 |
| --- | --- | --- |
| HQ 管理员 | 总部平台管理人员 | 管理全局资源、规则、权限和看板，掌握全球资源质量 |
| 区域管理员 | 各市场/区域负责人 | 管理本区域资源、项目、合作记录和数据更新 |
| 项目负责人 | 具体营销项目 Owner | 创建项目需求，获取推荐资源，确认合作名单并跟进结果 |
| 资源运营 | 资源维护人员 | 新增、编辑、更新资源信息，处理预警和数据补全 |
| 管理层 | 业务管理人员 | 查看整体资源资产、项目表现、风险和趋势 |
| 普通成员 | 查询和使用资源的业务人员 | 查询资源、提交资源、参与项目协作 |
| 系统管理员 | 平台技术/权限管理员 | 管理账号、角色、菜单、系统配置和操作日志 |

### 2.2 权限原则

- HQ 管理员默认可查看和管理全部资源、项目、规则和看板。
- 区域管理员默认可管理所属区域资源和项目，可查看全局资源摘要。
- 项目负责人可创建和管理本人负责的项目，可将可见资源加入候选名单。
- 资源运营可维护授权范围内的资源主档、账号数据、标签和预警处理结果。
- 管理层以只读看板和报表为主，不默认具备编辑权限。
- 普通成员可查看授权范围内资源，可提交资源线索，但不默认具备规则配置权限。
- 系统管理员负责用户、角色、菜单、权限和系统参数，不默认参与业务评分决策。

MVP 阶段若区域数据隔离暂未完全实现，默认以菜单权限和角色权限为主；区域隔离作为后续增强项，但 PRD 中的数据结构需预留区域字段。

## 5. 核心业务流程

### 5.1 资源入库流程

```text
新增资源
→ 填写基础信息
→ 系统校验必填字段
→ 打标签
→ 填写平台账号数据
→ 标记数据来源可信度
→ 系统计算评分
→ 生成资源等级
→ 进入资源库
```

关键规则：

- 必填字段缺失时，不允许保存为“可合作”；系统应标记为“待补全”或阻止提交，具体由必填规则配置决定。
- 资源保存后需计算资料完整度。
- 资源进入“待补全”“观察中”“暂停合作”“黑名单”“已归档”状态后，不默认参与智能推荐。
- 系统按名称、联系方式、平台账号 URL 等字段进行疑似重复提示。

### 5.2 智能推荐流程

```text
用户输入项目需求
→ AI 解析需求
→ 生成结构化条件
→ 按治理规则过滤资源
→ 按评分和匹配度排序
→ 生成推荐理由和风险提示
→ 输出推荐清单
→ 用户加入项目候选名单
```

AI 与规则分工：

- AI 负责理解自然语言需求、抽取结构化条件、生成解释文案。
- 系统规则负责过滤、排序、权限控制、预算约束和状态约束。
- AI 不直接决定最终可推荐资源，不绕过黑名单、权限和治理规则。
- AI 不可用时，系统应降级到本地规则推荐，并提示用户当前为规则推荐结果。

### 5.3 项目合作流程

```text
创建项目
→ 输入项目需求
→ 智能助手推荐资源
→ 加入候选名单
→ 确认合作资源
→ 跟进合作状态
→ 内容发布
→ 回填曝光/点击/转化/成本
→ 生成项目复盘
→ 更新资源评分
```

关键规则：

- 候选资源不等于已确认合作资源。
- 只有已确认合作的资源才要求后续回填合作记录。
- 合作记录回填后，应触发资源评分更新或生成待更新评分任务。
- 项目完成前，系统应提示未回填效果数据的合作资源。

### 5.4 治理闭环

```text
配置治理规则
→ 校验资源数据
→ 计算评分等级
→ 控制智能推荐
→ 触发预警
→ 推动资源更新
→ 优化规则
```

关键规则：

- 治理规则修改需保存版本。
- 评分权重、等级阈值、推荐规则变更后，应支持立即生效或定时生效。
- 规则生效后影响资源评分、等级、推荐资格或预警状态时，需记录影响范围。

## 6. 功能需求

### 6.1 首页工作台

#### 6.1.1 目标

首页用于展示当前用户最关心的信息，让用户快速知道现在有哪些资源、哪些项目在进行、哪些问题需要处理。

#### 6.1.2 入口

- 菜单：资源运营 > 首页工作台
- 登录后默认进入首页工作台，若用户无权限则进入第一个有权限菜单。

#### 6.1.3 展示内容

| 区域 | 字段/内容 |
| --- | --- |
| 指标卡 | 全球资源总数、本区域资源数量、S/A 级资源数量、待更新资源数量 |
| 项目概览 | 近期项目、进行中项目、待复盘项目、预算使用概况 |
| 预警待办 | 数据缺失、数据过期、评分下降、履约风险、联系失效 |
| 智能助手入口 | 输入框快捷入口，支持跳转到智能资源助手并带入需求 |
| 最近资源 | 最近新增资源、最近更新资源 |
| 推荐资源 | 高价值资源推荐、待复用资源 |

#### 6.1.4 交互规则

- 指标卡按当前用户数据权限展示。
- 点击指标卡可跳转到对应列表并带入筛选条件。
- 待处理预警按严重程度和更新时间排序。
- 智能助手快捷输入为空时，点击生成需提示输入项目需求。

#### 6.1.5 异常状态

- 无资源数据时，展示空状态和“新增资源”入口。
- 无项目数据时，项目概览展示空状态。
- 数据加载失败时，展示错误提示和重试按钮。

#### 6.1.6 验收标准

- 用户可在首页看到资源、项目、预警相关核心指标。
- 点击待更新资源可跳转资源库并筛选待更新资源。
- 输入项目需求后可跳转智能助手并自动带入输入内容。

### 6.2 智能资源助手

#### 6.2.1 目标

智能资源助手用于理解用户的自然语言项目需求，并从资源库中推荐合适的媒体、KOL 或创作者，输出可解释、可筛选、可加入项目的推荐清单。

#### 6.2.2 入口

- 菜单：资源运营 > 智能资源助手
- 首页工作台快捷入口
- 项目详情页“生成推荐”

#### 6.2.3 用户输入

| 输入项 | 必填 | 说明 |
| --- | --- | --- |
| 项目需求文本 | 是 | 自然语言描述目标市场、产品、预算、平台、行业、目标等 |
| 目标市场 | 否 | 可手动补充或覆盖 AI 解析结果 |
| 语言 | 否 | 可手动补充或覆盖 AI 解析结果 |
| 资源类型 | 否 | 媒体、KOL、创作者、代理商等 |
| 平台 | 否 | YouTube、TikTok、Instagram、媒体网站等 |
| 预算 | 否 | 项目总预算或单资源预算上限 |
| 投放目标 | 否 | 曝光、点击、转化、下载、注册等 |
| 是否包含观察资源 | 否 | 默认否 |

示例输入：

> 我们要在美国推广一款 AI 录音笔，预算 35000 美元，想找英语科技媒体和 YouTube 评测创作者，目标是新品曝光和点击转化。

#### 6.2.4 AI 解析输出

| 识别项 | 示例 |
| --- | --- |
| 目标市场 | 美国 |
| 语言 | 英语 |
| 行业 | AI、科技、消费电子 |
| 资源类型 | 媒体、YouTube 创作者 |
| 预算 | 35000 美元 |
| 投放目标 | 新品曝光、点击转化 |

#### 6.2.5 推荐结果字段

| 字段 | 说明 |
| --- | --- |
| 资源名称 | 媒体、KOL 或创作者名称 |
| 类型 | 媒体、KOL、创作者、代理商 |
| 平台 | YouTube、TikTok、Instagram、媒体网站等 |
| 国家/地区 | 主要市场 |
| 语言 | 内容语言 |
| 等级 | S/A/B/C/D |
| 综合评分 | 0-100 |
| 预估费用 | 资源预估合作费用 |
| 匹配度 | 系统计算的推荐匹配程度 |
| 推荐优先级 | 高、中、低 |
| 推荐理由 | 命中市场、行业、平台、历史效果等原因 |
| 命中规则 | 命中的推荐规则 |
| 降权原因 | 风险、数据过期、历史表现一般等原因 |
| 过滤原因 | 未入选时展示，如黑名单、预算超限、资料不完整 |
| 风险提示 | 高风险标签、履约风险、数据异常等 |

#### 6.2.6 硬条件过滤

- 国家/地区匹配。
- 语言匹配。
- 平台匹配。
- 行业领域匹配。
- 合作状态为可合作。
- 不在黑名单。
- 不为已归档资源。
- 报价不超过预算上限。
- 资料完整度达到推荐规则要求。
- 更新时间不超过推荐规则允许范围。
- 用户具备查看该资源的权限。

#### 6.2.7 软条件排序

- 综合评分越高越优先。
- 历史合作效果越好越优先。
- 互动率越高越优先。
- 数据可信度越高越优先。
- 风险越低越优先。
- 与投放目标越匹配越优先。
- 近期被采纳且表现稳定的资源可适当加权。
- 长期未更新或近期差评资源需降权。

#### 6.2.8 页面按钮

| 按钮 | 行为 |
| --- | --- |
| 生成推荐 | 解析需求并生成推荐结果 |

#### 6.2.9 异常状态

- 需求文本为空时，不允许生成推荐。
- AI 服务不可用时，降级到本地规则推荐，并提示“当前为规则推荐结果”。
- 无匹配资源时，展示过滤原因统计和放宽条件建议。
- 预算不足时，展示预算不足提示和低预算备选资源。
- 用户无项目权限时，不展示加入项目按钮或提示无权限。

#### 6.2.10 验收标准

- 用户输入示例需求后，系统能识别市场、语言、行业、资源类型、预算和目标。
- 推荐结果不包含黑名单、已归档、资料不完整且不允许推荐的资源。
- 推荐列表展示推荐理由、风险提示和命中规则。
- 用户可选择资源加入项目候选名单。
- AI 不可用时，系统仍可返回基于规则的推荐结果。

### 6.6 评分评级

#### 6.6.1 目标

评分评级用于判断资源质量，并影响筛选、排序、推荐、预警和资源运营优先级。

#### 6.6.2 默认评分维度

| 维度 | 默认权重 | 说明 |
| --- | ---: | --- |
| 影响力 | 20 | 粉丝量、媒体流量、账号覆盖能力 |
| 活跃度 | 15 | 近 30/90 天内容发布频率 |
| 互动质量 | 20 | 点赞、评论、分享、互动真实性 |
| 品牌契合 | 15 | 行业、内容风格、受众是否匹配 |
| 履约表现 | 20 | 合作准时率、沟通效率、交付质量 |
| 转化效果 | 10 | 点击、注册、下载、购买等结果 |

#### 6.6.3 默认评级规则

| 等级 | 分数区间 | 说明 |
| --- | ---: | --- |
| S | >= 90 | 核心资源，重点维护 |
| A | >= 80 | 优质资源，优先推荐 |
| B | >= 65 | 常规可用 |
| C | >= 50 | 观察资源 |
| D | < 50 | 谨慎使用或不推荐 |

#### 6.6.4 功能操作

- 自动计算评分。
- 支持手动调整评分。
- 查看评分原因。
- 查看评分历史。
- 规则变更后重新计算评分。
- 资源等级变化提醒。

#### 6.6.5 交互规则

- 综合评分默认由各维度分和权重计算得出。
- 手动调整评分必须填写调整原因。
- 评分规则修改后，应支持模拟影响范围。
- 规则生效后，系统自动更新资源等级。
- 资源评分下降超过预警阈值时，触发评分下降预警。

#### 6.6.6 验收标准

- 系统可以根据默认维度计算综合评分。
- 用户可以查看评分原因和评分历史。
- 修改等级阈值后，资源等级按新规则更新。
- 手动调整评分会记录调整人、时间和原因。

#### 6.8.4 评分影响规则

- 支持保存版本。

#### 6.10.5 必填数据规则

| 资源类型 | 默认必填字段 |

系统动作：

#### 6.10.6 更新频率规则

| 资源等级 | 默认更新要求 |
| --- | --- |
| S/A 级资源 | 每 30 天复核一次 |
| B/C 级资源 | 每 90 天复核一次 |
| D 级资源 | 每 180 天复核一次或归档 |

系统动作：

- 到期提醒负责人。
- 超期标记为待更新。
- 长期未更新降低推荐权重。

#### 6.10.7 数据可信度规则

| 可信度 | 来源 | 评分参与系数 |
| --- | --- | ---: |

| B | 创作者后台截图或录屏 | 90% |
| C | 第三方工具估算 | 80% |
| D | 人工公开页面采集 | 70% |

要求：

- D 级数据可要求人工复核。
- 数据可信度参与评分和推荐排序。
- 数据来源缺失时，默认按低可信度处理。

#### 6.10.8 智能推荐规则

| 规则项 | 默认策略 |
| --- | --- |

| 黑名单过滤 | 黑名单永不推荐 |
| 观察资源处理 | 默认不推荐，除非手动勾选 |
| 数据完整度 | 低于 80% 不推荐 |
| 更新时间 | 超过 180 天未更新不推荐 |
| 预算约束 | 报价超过预算上限的不推荐 |
| 风险标签 | 高风险标签降权或过滤 |
| 历史表现 | 近 3 次合作差评降权 |

推荐结果需要展示：

- 推荐理由。
- 命中规则。
- 降权原因。
- 过滤原因。
- 风险提示。

#### 6.10.9 预警规则

| 预警类型 | 默认触发条件 |
| --- | --- |
| 数据缺失预警 | 必填字段缺失 |
| 数据过期预警 | 超过复核周期未更新 |
| 评分下降预警 | 较上次下降超过 10 分 |
| 互动异常预警 | 互动率异常偏高或偏低 |
| 成本异常预警 | 报价高于同类均值 50% |

预警动作：

- 站内通知。
- 标记资源状态。
- 推送给负责人。
- 降低推荐权重。
- 加入观察池。

## 7. 资源状态设计

| 状态 | 说明 | 是否参与推荐 |
| --- | --- | --- |
| 可合作 | 正常资源 | 是 |
| 待补全 | 资料不完整 | 否 |
| 观察中 | 存在一定风险 | 默认否 |
| 暂停合作 | 临时不可用 | 否 |
| 黑名单 | 严重风险资源 | 否 |
| 已归档 | 历史资源 | 否 |

自动流转示例：

- 必填字段缺失 -> 待补全。
- 超 180 天未更新 -> 观察中。
- 连续 2 次延期交付 -> 观察中。
- 严重舆情风险 -> 暂停合作。
- 确认刷量或欺诈 -> 黑名单。
- 长期无效 -> 已归档。

状态变更规则：

- 自动流转需记录触发规则。
- 手动流转需记录操作人、时间和原因。
- 从黑名单恢复为可合作需 HQ 管理员权限。
- 已归档资源默认不展示在普通查询结果中，可通过筛选查看。

## 9. 技术与集成要求

### 9.1 前端

- 技术栈：vue-pure-admin。
- 表格、筛选、详情页、配置表单为核心组件。
- 看板可使用 ECharts 或现有图表能力。
- 敏感字段按权限脱敏展示。
- 推荐结果、规则命中、过滤原因需具备可读性。

### 9.2 后端

- 技术栈：Go + MySQL。
- 推荐服务需封装为后端能力，前端只调用业务接口。
- 规则过滤、权限校验、黑名单过滤必须在后端执行。
- AI 调用失败时，应返回可识别错误码并支持规则降级。
- 操作日志、规则版本、预警记录需后端统一写入。

### 9.3 数据库

- MySQL 存储业务数据。
- Redis 可用于缓存权限、规则配置、热点看板数据。
- Elasticsearch/OpenSearch 可作为后续复杂搜索能力，不作为 MVP 必须依赖。
- 对象存储可用于后续保存截图、报价单、数据证明材料，不作为 MVP 必须依赖。

### 9.4 AI 助手服务

推荐流程：

```text
用户输入
→ 大模型提取需求
→ 本地规则过滤
→ 数据库查询资源
→ 推荐算法排序
→ 大模型生成解释
→ 返回推荐结果
```

技术要求：

- AI Key 仅后端保存，加密存储。
- AI 输出必须经过后端结构校验。
- AI 不得绕过用户权限和资源状态规则。
- AI 解析失败时，允许用户手动补充结构化条件。
- AI 推荐解释不得输出用户无权限查看的敏感字段。

## 10. 非功能需求

### 10.1 性能

- 资源列表查询在常规筛选条件下，目标响应时间不超过 2 秒。
- 智能推荐在资源量较小时目标响应时间不超过 10 秒；超时需提示用户稍后重试或展示规则推荐。
- 看板默认时间范围查询目标响应时间不超过 3 秒。
- 批量导入需支持异步处理或进度提示，避免页面长时间无反馈。

### 10.2 可用性

- 表单字段需有清晰必填标识。
- 规则配置页需提供恢复默认和保存前校验。
- 高风险操作需二次确认。
- 导入失败需展示行号、字段和错误原因。
- 推荐无结果需展示过滤原因统计和放宽条件建议。

### 10.3 数据质量

- 必填字段由治理规则控制。
- 资料完整度由系统计算。
- 数据来源和采集时间需作为平台账号关键字段。
- 资源重复需提示，不应静默创建大量重复资源。
- 合作效果回填率需纳入看板指标。

### 10.4 安全

- API Key 加密存储，前端不展示完整 Key。
- 联系方式可按权限脱敏。
- 黑名单原因、风险备注等敏感信息需按角色控制查看。
- 所有关键操作记录操作日志。
- 推荐接口必须进行权限校验，避免越权查看资源。

## 11. 验收标准

### 11.1 产品验收

- MVP 范围明确包含智能资源助手、资源库、标签、评分评级、项目管理、合作记录、治理规则和基础看板。
- Brief 模板、素材包、邮件生成、自动合同、自动付款、深度内容审核、全平台自动采集不作为 MVP 必做项。
- 每个核心模块均包含目标、入口、字段、操作、规则、异常状态和验收标准。
- 业务闭环覆盖资源入库、智能推荐、项目合作、效果回填、评分更新和治理预警。

### 11.2 研发验收

- 资源主档和平台账号支持一对多关系。
- 智能助手推荐结果经过后端规则过滤，不包含黑名单、已归档和无权限资源。
- 治理规则支持版本化保存。
- 评分权重、等级阈值、必填字段、更新频率、推荐规则和预警规则可配置。
- 项目候选资源和合作记录可以关联资源与项目。
- 看板指标可从资源、项目、合作记录、预警数据中计算。

### 11.3 测试验收场景

| 场景 | 预期结果 |
| --- | --- |
| 新增资源缺少必填字段 | 系统提示缺失字段，资源标记为待补全或阻止保存 |
| 黑名单资源参与推荐 | 推荐结果中不出现该资源 |
| 观察中资源默认推荐 | 默认不出现，用户勾选允许观察资源后可按规则出现 |
| 资源超过更新周期 | 生成待更新预警，并降低推荐权重或不推荐 |
| AI 服务不可用 | 系统降级到规则推荐并提示用户 |
| 输入美国 AI 录音笔推广需求 | 系统识别美国、英语、AI/科技、媒体/YouTube、35000 美元、曝光/点击转化 |
| 推荐资源加入项目 | 资源进入项目候选名单，状态为候选 |
| 合作发布后回填效果数据 | 合作记录更新，项目看板和资源评分可刷新 |
| 修改评分权重 | 保存新规则版本，可查看影响资源范围 |
| 修改等级阈值 | 资源等级按新阈值重新计算 |
| 导入重复资源 | 系统提示疑似重复，不静默创建重复档案 |
| 联系方式无效 | 生成联系失效预警并通知负责人 |

## 12. 建设路线图

### 12.1 第一阶段：0-3 个月，MVP 上线

目标：把资源管起来，让智能助手能推荐资源。

建设内容：

- 资源库。
- 标签体系。
- 评分模型。
- 智能助手基础推荐。
- 项目管理。
- 合作记录。
- 治理规则基础配置。
- 基础看板。

### 12.2 第二阶段：3-6 个月，规则治理完善

目标：提升数据质量和管理效率。

建设内容：

- 必填字段规则。
- 更新频率规则。
- 数据可信度规则。
- 推荐规则。
- 预警规则。
- 规则版本管理。
- 资源状态自动流转。

### 12.3 第三阶段：6-12 个月，智能化增强

目标：提升推荐准确度和复盘能力。

建设内容：

- AI 推荐解释优化。
- 推荐采纳率统计。
- 预算分配建议增强。
- 项目复盘自动总结。
- 资源风险识别。

## 14. 最终业务闭环

平台最终形成以下闭环：

```text
治理规则配置
→ 资源入库
→ 标签化管理
→ 评分评级
→ 智能助手推荐
→ 加入项目
→ 合作执行
→ 效果回填
→ 资源评分更新
→ 预警与状态流转
→ 规则持续优化
```

> 让全球媒体/KOL资源从分散资料变成企业可运营、可评估、可复用、可增长的数据资产。
